# Seed Data Enrichment Plan

**Status:** Plan — pending execution.
**Last updated:** 2026-04-19.
**Related:** [`docs/architecture/ROOT_SIGNAL_DATA_CONTRACT.md`](../architecture/ROOT_SIGNAL_DATA_CONTRACT.md) — the shape we're enriching toward.

---

## Goals

1. **Every seed post should render a complete detail page.** Today, a post like `Highway 169 Bridge Deck Work` has `body_raw = bodyLight = 100 chars` and no sidebar data. A reader landing on its detail page sees a headline and one sentence.
2. **Exercise every field group at least ~5× across the corpus.** Today some groups (`post_contacts`, individual-source posts) are used zero times. UI bugs hide in unexercised paths.
3. **Test the full Root Signal data contract, not a subset.** If Signal will eventually produce X, our seed should already contain X shaped the same way.
4. **Approximate a real weekly pool per county.** Enough posts across weight classes and post_types that the layout engine's three-phase selection has realistic inputs.

## Non-goals

- Not preserving existing post IDs (user confirmed dummy data; IDs can change).
- Not a content-moderation pass (no fact-checking against real MN events).
- Not building the media ingest pipeline. Placeholder images from `data/seed-media/` are fine.
- Not introducing new schema migrations beyond what's already landed through 232.

## Strategy — three passes + one scaffolding step

### Pass 0 — Infrastructure (seed.mjs gaps)

Before enriching content, the seed script itself needs to cover field groups it currently ignores:

- **`post_contacts`** — seed.mjs does not emit rows. Add a handler that reads a `contacts: [{type, value, label}]` array from post JSON.
- **`posts.organization_id`** — set directly from post JSON (`organizationName` lookup) rather than the roundabout `post_source_attribution → organizations → sources → post_sources` join.
- **Individual sources (`source_individuals`)** — table doesn't exist yet. Add migration + seed handler. Deferred to its own sub-step so we can land Passes 1–3 independently.
- **Validator** — extend `data/audit-seed.mjs` to run as part of `make seed` and fail loudly if gap count regresses.

### Pass 1 — Body tiers (fixes ~140 gaps)

Target: bring every post above the body contract floors.

- For every post where `body_raw < 250`, write a full body (target 400–800 chars) using the existing title + `body_medium`/`body_light` as seeds. For `weight=light` updates (40 posts), this is the bulk of the work.
- Fill `body_medium` on the 50 posts missing it. Target 200 chars.
- Fill `body_heavy` on any remaining heavy/medium posts where it's empty. Target 800 chars for heavy, 400 for medium.

This pass is purely text generation; every edit happens in `data/posts.json`.

### Pass 2 — Sidebar richness (fixes ~200 gaps)

Target: every post gets the field groups §7 requires for its type.

Per post_type:

| post_type | What to add | Count to fix |
|---|---|---|
| `update` | Add `link` or `contacts` (source URL or a phone/website), add `sourceAttribution`, add `meta.byline` | 40 |
| `event` | Add `link` (register/more info) or `contacts`, add `sourceAttribution` | 26 |
| `need` | Add `items` (what's needed, with qty/urgency), add `contacts` (point of contact), add `status` | 8 |
| `aid` | Add `items` (what's available), `contacts`, `status` | 21 |
| `reference` | Add `contacts` per directory, ensure all have `items`, mark `is_evergreen` | 21 |
| `business` | Add `contacts` (phone/address/website), `scheduleEntries`, link to org | 9 |
| `person` | Add `media` (hero photo), ensure `sourceAttribution` | 10 |
| `story` | Add `meta.byline` + `meta.deck` where missing; heavy stories get a hero image | varies |
| `action` | Ensure `link.deadline` is set for time-bound actions | small |

### Pass 3 — Source graph (fixes dedup + individual-source coverage)

Target: turn flat posts into a graph with real source attribution.

- **Expand `data/organizations.json`** from 50 → ~70 orgs. Add state agencies (MnDOT, MDH, DEED, DHS, MN Housing, MN Attorney General), a few state-level nonprofits (MN Community Action Partnership, Greater Twin Cities United Way), and broaden county coverage beyond Dakota. Each org: name, website, phone, address, populations_served, county, social handles where known.
- **Link ~60% of posts to an organization** via `organizationName` in post JSON. The remaining 40% stay editorial (Root Editorial Staff byline) or become individual-source posts.
- **Introduce ~15 individual-source posts** (Instagram/Facebook/X content) once the `source_individuals` table lands. Mix of identity-verified and consent-pending states so the editor review queue has realistic content.
- **Add `post_sources` rows with real `source_url` values** for every Signal-sourced post. These double as the dedup graph Signal will use in production.

## Pass 0 deliverables (concrete)

1. Extend `data/seed.mjs` to emit `post_contacts` rows from a `contacts` array on post JSON.
2. Extend `data/seed.mjs` to resolve `organizationName` on post JSON → `organizations.name` → set `posts.organization_id` directly. Keep the existing `post_sources` join for belt-and-suspenders.
3. Add `make audit-seed` target that runs `node data/audit-seed.mjs` and exits non-zero when gap counts regress against a committed baseline (`data/audit-seed.baseline.json`).
4. Document the JSON shape extensions in `data/posts.json` with a comment block at the top.

Individual-source table and ingest path remain their own separate work — not required to unblock Passes 1–3. Pass 3's individual-source subtask stays parked until the schema lands.

## Pass 1 deliverables

1. Rewrite the body fields of all 92 posts currently below `body_raw` ≥ 250.
2. Fill all 50 missing `body_medium` fields.
3. Fill all ~12 missing `body_heavy` fields on heavy-weight posts.
4. Run `node data/audit-seed.mjs` — expected gap reduction: ~150 gaps → 0 on body-related rules.

Quality bar per body:

- `body_light` stays single-sentence, 30–120 chars. Can be kept as-is where it already is.
- `body_medium` (150–280): 2 sentences. Covers who, what, when, where. One actionable detail.
- `body_heavy` (600–1400): 3–5 paragraphs. Full context; links to source; named sources where available.
- `body_raw` (≥250, typically matches `body_heavy`): the full editorial body for the detail page.

No fake quotes. If a story needs a quote to feel real, mark the byline as "Root Editorial Staff" and attribute the quote to a named role ("a county spokesperson said"), not a fabricated individual.

## Pass 2 deliverables

1. Add `meta.byline` to all 106 posts missing it.
2. Add `sourceAttribution` (`sourceName` + `attribution`) to all 110 posts missing it.
3. Add per-post-type required groups:
   - `update`/`event`: link or contacts (40 + 26)
   - `aid`: items + contacts + status (21)
   - `reference`: contacts per directory entry (21)
   - `need`: items + contacts + status (8)
   - `business`: contacts + scheduleEntries (9 + 2)
   - `person`: media (10)
4. Run audit — expected: `type_group:*` gaps → 0, `source:*` → 0, `meta:byline_missing` → 0.

## Pass 3 deliverables

1. Add ~20 new organizations to `data/organizations.json` (state agencies + broader county coverage).
2. Add `organizationName` to ~100 of the 168 posts (≈60%). For each, the name must match an entry in `organizations.json`.
3. Add `sourceUrl` on every `sourceAttribution`-bearing post. Real URLs where possible; `https://example.mn.gov/...` placeholders otherwise.
4. Pull in ~20–30 *additional* net-new posts that exercise underrepresented shapes:
   - 5 individual-source Instagram posts (pending `source_individuals` schema — see Pass 0 note).
   - 3 posts in `in_review` status (low extraction_confidence; `consent_to_publish=false`) so the editor queue isn't empty.
   - 3 revision chains (`revision_of_post_id`) so admin revision UI has something to render.
   - 2 archived posts (older than 30 days).
   - 5 evergreen references with 10+ items each so the directory template has dense content to stress.
   - 2 truly metro-scope posts with multiple `service_area` tags.

---

## Success criteria

- Audit reports **zero** gaps on the body-tier, meta, source-attribution, and type-group rules.
- Every post_type has ≥3 examples that exercise its required field groups.
- At least 5 posts exist with every optional field populated (exemplars for demos).
- `make seed && make audit-seed` passes on a clean DB without manual intervention.
- A reader can land on any post's detail page and get a complete experience: body, sidebar (schedule/contacts/link/media), and source attribution.
