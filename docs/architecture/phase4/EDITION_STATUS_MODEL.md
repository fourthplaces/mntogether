## Statuses

| Status | Meaning | Who triggers |
|--------|---------|-------------|
| `draft` | Auto-generated by Root Signal / layout engine, untouched by editors | System (batch_generate) |
| `in_review` | Editor has opened the broadsheet in the layout editor | Auto on open |
| `approved` | Editor has signed off, ready for publication | Editor (approve button) |
| `published` | Live on the public site | Editor (publish / "Approve All" button) |
| `archived` | Taken down from public site | Editor |

## Transition Rules

- `draft ‚Üí in_review`: Auto-triggered when editor opens broadsheet editor
- `in_review ‚Üí approved`: Explicit editor action (approve button)
- `approved ‚Üí published`: Explicit editor action, or batch via "Approve All" button
- `draft ‚Üí published`: Allowed (backward compat ‚Äî skip review for quick publish)
- `published ‚Üí archived`: Explicit editor action

Invalid transitions return errors. The SQL WHERE clause enforces source-state preconditions atomically.

## Batch Operations

- **batch_approve**: Transitions multiple `in_review` ‚Üí `approved`. Skips editions not in `in_review`.
- **batch_publish**: Transitions multiple `approved` ‚Üí `published`. Powers the "Approve All" button on the kanban board.

## Kanban Stats Query

`editionKanbanStats(periodStart, periodEnd)` returns counts grouped by status for the given week:

```graphql
type EditionKanbanStats {
  draft: Int!
  inReview: Int!
  approved: Int!
  published: Int!
}
```
