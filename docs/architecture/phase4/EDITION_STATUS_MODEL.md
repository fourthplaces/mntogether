## Status State Machine

```
draft ──[editor opens broadsheet]──→ in_review ──[approve]──→ approved ──[publish]──→ published
                                                                                         │
                                                                                    [archive]
                                                                                         ↓
                                                                                     archived
```

## Statuses

| Status | Meaning | Who triggers |
|--------|---------|-------------|
| `draft` | Auto-generated by Root Signal / layout engine, untouched by editors | System (batch_generate) |
| `in_review` | Editor has opened the broadsheet in the layout editor | Auto on open |
| `approved` | Editor has signed off, ready for publication | Editor (approve button) |
| `published` | Live on the public site | Editor (publish / "Approve All" button) |
| `archived` | Taken down from public site | Editor |

## Transition Rules

- `draft → in_review`: Auto-triggered when editor opens broadsheet editor
- `in_review → approved`: Explicit editor action (approve button)
- `approved → published`: Explicit editor action, or batch via "Approve All" button
- `draft → published`: Allowed (backward compat — skip review for quick publish)
- `published → archived`: Explicit editor action

Invalid transitions return errors. The SQL WHERE clause enforces source-state preconditions atomically.

## Batch Operations

- **batch_approve**: Transitions multiple `in_review` → `approved`. Skips editions not in `in_review`.
- **batch_publish**: Transitions multiple `approved` → `published`. Powers the "Approve All" button on the kanban board.

## Kanban Stats Query

`editionKanbanStats(periodStart, periodEnd)` returns counts grouped by status for the given week:

```graphql
type EditionKanbanStats {
  draft: Int!
  inReview: Int!
  approved: Int!
  published: Int!
}
```
