## Overview

The workflow page (`/admin/workflow`) has been rewritten from a post-level kanban to an **edition-level kanban board**. Editors now review and manage 87 weekly broadsheets (one per county) through a 4-column board.

## Columns

| Column | Edition Status | Color | Description |
|--------|---------------|-------|-------------|
| Live | `published` | Green | Currently on the public site |
| Ready for Review | `draft` | Warm beige | Auto-generated by Root Signal, untouched |
| In Review | `in_review` | Amber | Editor is actively reviewing/editing |
| Approved | `approved` | Sage | Signed off, ready for publication |

## Features

### Week Navigation
- Auto-selects current week (Monday–Sunday)
- Previous/next week arrows
- "This Week" reset button
- Period filters passed to backend via new `periodStart`/`periodEnd` query args

### Drag-and-Drop
- Uses `@dnd-kit/core` + `@dnd-kit/sortable` (same as old post-level kanban)
- Drag Ready for Review → In Review calls `reviewEdition`
- Drag In Review → Approved calls `approveEdition`
- Drag Approved → Live calls `publishEdition`
- Backward transitions not supported

### Batch Actions
- **Publish All** button in the Approved column header — batch publishes all approved editions via `batchPublishEditions` mutation

### Card Design
- County name (primary text)
- Period date range
- Row count indicator with fill bar
- Click to edit (links to `/admin/editions/[id]`)

## Navigation Changes

Sidebar reordered:
1. **Review Board** (was "Workflow" — moved to top of Content group)
2. Editions
3. Posts (with sub-menu)
4. Media

## Backend Changes

- Added `period_start` and `period_end` to `EditionFilters` and `ListEditionsRequest`
- Updated SQL queries in `Edition::list()` to filter by period
- GraphQL `editions` query now accepts `periodStart` and `periodEnd` args
